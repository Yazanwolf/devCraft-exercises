Approach to scaling database as Product Owner

1. Defining goals and constraints:
I would align with stakeholders on performance targets, budget, and timelines to establish clear success criteria.

2. Comparing both solutions
It make sense to evaluate vertical and horizontal scaling together with the team, considering cost, effort, and expected impact.

3. Contain immediate user impact if possible
Implement a short-term solution such as vertical scaling to quickly stabilize system performance, consider caching or temporarily workarounds.

4. Plan long-term scalability iteratively
Introduce horizontal scaling (sharding) step by step through backlog items, starting with a proof of concept.

5. Monitor and adapt
Continuously track system performance and user feedback, adjusting priorities as needed.

Conclusion
Focus on immediate stabilization while building a scalable long-term solution, using an iterative Scrum approach.
